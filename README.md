#  Веб-приложение "Книжный магазин"

Веб-приложение для управления книжным магазином с гибридным хранилищем данных. Включает каталог книг, систему отзывов, акции, расчет популярности жанров и кэширование статистики.

##  Технологический стек

*   Backend: Flask (Python 3)
*   Relational DB: PostgreSQL (основные данные: книги, жанры, заказы)
*   Document DB: MongoDB (отзывы, промо-акции)
*   Cache & Analytics: Redis (счетчики просмотров, кэш рейтингов, топ популярного)
*   ORM: SQLAlchemy (для PostgreSQL)
*   ODM: PyMongo (для MongoDB)

## Архитектура и основные фичи

1.  **Гибридное хранение:**
    *   PostgreSQL: Структурированные данные (`books`, `genres`, `order_history`).
    *   MongoDB: Динамические сущности (`reviews`, `promotions`).
    *   Синхронизация: При добавлении отзыва в Mongo автоматически обновляется усредненный `rating` книги в PostgreSQL.

2.  **Redis для высоконагруженных операций:**
    *   Счетчик просмотров книг.
    *   Кэширование среднего рейтинга.
    *   Хранение ранжированного списка популярных книг через `ZSET` (`popular:books`).

3.  **Система скидок:**
    *   Поддержка процентных скидок и акций "купи X, получи Y бесплатно".
    *   Акции привязываются к конкретной книге или целому жанру.

4.  **Динамический рейтинг жанров:**
    *   Модуль `m.py` автоматически пересчитывает популярность жанра на основе количества заказов за последние 60 дней.

##  Требования

*   Python: 3.8+
*   PostgreSQL: 12+
*   MongoDB: 4.4+
*   Redis: 6.0+

##  Установка и запуск
1. Клонирование репозитория
```bash
git clone <repository-url>
cd bookstore-project
```
2. Установка зависимостей
```bash
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate (для Windows)
pip install -r requirements.txt
```
3. Настройка окружения (.env)
Создайте файл .env в корне проекта на основе config.py

4. Запуск инфраструктуры (БД)
Убедитесь, что сервисы PostgreSQL, MongoDB и Redis запущены локально.

5. Первый запуск приложения
```bash
python app.py
```
